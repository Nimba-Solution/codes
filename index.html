Nouvelle regles de collaborations chez Nimba

Cycle standard de developement

  1. Le client ecrit ses besoins sous forme de scenario
  2. L’équipe transforme les scénarios en tâches à réaliser et en tests fonctionnels
  3. Les developeurs evaluent le cout de chaque scenario en collaboration avec le client
  4. Le client choisie les scenarios a integrer pour la prochaine livraision
  5. Chaque developeurs prends la responsabilite d’une tache pour la realisation d’un scenario
  6. Le developeurs choisit un binome
  7. Le binome ecrit les tests unitaires correspondants aux taches a implementer
  8. Le binome procede a l’implementation proprement dite
  9. Le developeur integre son travail a la version d’integration qui fera partir du prochain release.

Planification collective

  1. Un project est decompose en plusieurs iterations, chaque iterations prends une semaine ou deux selon le projet.
  2. Un rythme de livraison a partir de 2 iterations ou 3 en generale.
  3. Le cycle de developement porte sur la planification des fonctionalites decrites sous forme de scenairo (user stories)
  4. Le cycle des iterations porte sur la planificaiton des taches menees par les developeurs
  5. Les taches sont des activites des developeurs qui permettent de realiser un scenario client.

Discipline

  1. Communication
  2. La simplicite
  3. Feedback
  4. Courage
  5. Respect
  6. Convention de nommage, variable, function, class, fichier etc…
  7. Software craftsmanship
